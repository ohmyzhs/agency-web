# 이미지 포맷 컨버터 사용법: PNG·JPG·WebP·AVIF를 언제 선택할까

이미지 변환 도구를 열면 가장 먼저 막히는 지점은 버튼이 아니라 선택지다. PNG, JPG, WebP, AVIF 중 무엇을 골라야 하는지 애매하다. 파일 크기를 줄이고 싶지만 화질이 너무 깨지면 안 되고, 투명 배경은 유지해야 할 때도 있고, 블로그나 CMS가 특정 형식을 받지 않는 경우도 있다.

Image Format Converter는 이미지를 다른 포맷으로 바꾸고, 크기를 줄이고, 품질을 조절해 다운로드하는 도구다. 핵심은 “가장 작은 파일”을 만드는 것이 아니라 사용처에 맞는 형식을 고르는 것이다.

## 먼저 사용처를 정한다

이미지 변환은 목적 없이 시작하면 결과를 판단하기 어렵다. 같은 사진이라도 블로그 본문, 썸네일, 로고, 제출용 문서, 메신저 공유용 이미지에 따라 좋은 선택이 달라진다.

처음에는 다음 질문부터 정리한다.

- 웹 페이지에 올릴 이미지인가, 문서 제출용 이미지인가
- 투명 배경이 필요한가
- 사진인가, UI 캡처나 로고처럼 선과 글자가 많은 이미지인가
- 받는 서비스가 WebP나 AVIF를 지원하는가
- 원본을 보관해야 하는가, 변환본만 있으면 되는가

이 질문을 먼저 통과하면 포맷 선택이 훨씬 쉬워진다.

## PNG가 어울리는 경우

PNG는 선명한 그래픽과 투명 배경에 강하다. 로고, 아이콘, UI 캡처, 차트, 텍스트가 많은 이미지에는 PNG가 잘 맞는다. 손실 압축이 아니라서 반복 저장해도 JPG처럼 글자 가장자리가 흐려지는 문제가 적다.

다만 사진에는 파일이 커질 수 있다. 스마트폰 사진을 PNG로 바꾸면 용량이 오히려 커질 수 있다. 그래서 PNG는 “선명함과 투명도”가 필요한 경우에 쓰고, 일반 사진 압축에는 다른 형식을 먼저 생각하는 편이 좋다.

PNG를 고르면 좋은 상황은 다음과 같다.

- 투명 배경이 필요한 로고나 아이콘
- UI 스크린샷, 코드 캡처, 문서 캡처
- 색 영역이 단순하고 선이 중요한 그래픽
- 편집 원본에 가까운 보존용 이미지

## JPG가 어울리는 경우

JPG는 사진에 강하다. 풍경, 인물, 음식, 제품 사진처럼 색이 자연스럽게 이어지는 이미지에서 파일 크기를 줄이기 좋다. 대부분의 서비스가 지원하고, 오래된 시스템과의 호환성도 좋다.

하지만 JPG는 투명 배경을 유지하지 못한다. 투명한 PNG를 JPG로 내보내면 배경색을 채워야 한다. 이때 배경색을 흰색으로 할지, 사이트 배경색에 맞출지 확인해야 한다. 또한 품질 값을 너무 낮추면 글자와 경계가 뭉개진다.

JPG를 쓸 때는 다음을 확인한다.

- 투명 배경이 필요 없는가
- 사진 중심 이미지인가
- 품질 값이 너무 낮아 글자가 흐려지지 않는가
- 배경색을 채워도 어색하지 않은가
- 원본은 따로 보관했는가

## WebP와 AVIF는 웹 최적화용으로 본다

WebP와 AVIF는 웹에서 파일 크기를 줄이기 위해 자주 쓰인다. 특히 블로그나 도구 페이지에 올릴 이미지는 WebP만으로도 체감 용량을 줄일 수 있다. AVIF는 더 작은 파일을 만들 수 있는 경우가 있지만, 브라우저와 편집 도구, CMS 지원 차이를 확인해야 한다.

실무적으로는 WebP를 먼저 시도하고, 지원 환경이 확실할 때 AVIF를 검토하는 흐름이 안전하다. 이미지가 안 보이면 파일이 작아진 의미가 없기 때문이다. 중요한 대표 이미지는 PNG 또는 JPG 원본을 보관하고, 웹 배포용 사본으로 WebP나 AVIF를 만든다.

WebP/AVIF를 쓸 때의 기준은 다음과 같다.

- 웹 페이지 성능을 위해 용량을 줄이고 싶다.
- 대상 브라우저나 플랫폼이 해당 형식을 지원한다.
- 원본 이미지 파일을 별도로 보관한다.
- 변환 후 실제 페이지에서 이미지가 보이는지 확인한다.

## 크기 조절은 품질 조절보다 먼저다

이미지 용량을 줄일 때 품질 슬라이더만 낮추는 경우가 많다. 하지만 원본 해상도가 지나치게 크다면 먼저 크기를 줄이는 편이 더 자연스럽다. 예를 들어 본문 폭이 800px인 블로그에 4000px 사진을 그대로 올릴 필요는 없다.

작업 순서는 보통 이렇다.

1. 사용처에 필요한 최대 너비와 높이를 정한다.
2. 원본 비율을 유지한 채 크기를 줄인다.
3. 포맷을 선택한다.
4. JPG/WebP/AVIF라면 품질 값을 조정한다.
5. 다운로드 후 실제 화면에서 확인한다.

크기와 품질을 동시에 과하게 낮추면 이미지가 흐릿해진다. 먼저 치수를 맞추고, 그 다음 압축 강도를 조절하는 편이 결과를 읽기 쉽다.

## 배치 변환에서는 파일명을 신경 쓴다

여러 이미지를 한 번에 변환할 때는 파일명 관리가 중요하다. 변환 결과가 모두 `image.webp`처럼 나오면 나중에 어떤 파일이 어느 글에 쓰였는지 찾기 어렵다.

좋은 파일명은 용도를 드러낸다. 예를 들어 `pdf-toolkit-workflow-og.webp`, `favicon-source-512.png`, `blog-post-screenshot-01.webp`처럼 slug나 사용처를 넣으면 관리가 쉽다. 일괄 다운로드 ZIP을 받을 때도 변환 전후 파일명이 대응되어야 한다.

## 브라우저 로컬 처리의 의미

Image Format Converter는 Canvas와 ZIP 생성을 이용해 브라우저 안에서 처리하는 방향의 도구다. 사진이나 브랜드 이미지가 불필요하게 서버로 올라가지 않는다는 점이 장점이다.

하지만 브라우저 로컬 처리는 기기 성능의 영향을 받는다. 아주 큰 이미지나 많은 파일을 한 번에 처리하면 모바일 브라우저가 느려질 수 있다. 그래서 파일 수와 용량 제한을 두고, 실패하면 더 작은 묶음으로 나눠서 작업하는 편이 좋다.

로컬 처리라도 운영 비밀값, 고객 신분증, 민감한 내부 문서 이미지는 조심해서 다뤄야 한다. 도구가 서버로 보내지 않는 것과 사용자의 화면·다운로드 폴더 관리가 안전한 것은 별개의 문제다.

## 마지막 확인 체크리스트

변환 후에는 아래 항목을 확인한다.

- 사용처가 선택한 포맷을 지원하는가
- 투명 배경이 의도대로 유지되거나 채워졌는가
- 모바일 화면에서도 글자와 경계가 흐리지 않은가
- 파일 크기가 충분히 줄었는가
- 원본 파일을 따로 보관했는가
- 웹 페이지나 CMS에 실제로 업로드해 보였는가

이미지 포맷 컨버터는 한 번에 정답을 찍는 도구가 아니다. 사용처와 제약을 확인하고, 적절한 형식과 크기를 고르는 도구다. 이 순서만 지켜도 블로그와 도구 페이지의 이미지 품질을 더 안정적으로 관리할 수 있다.
